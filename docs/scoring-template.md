# 妙味スコアリング手順書（秘書用・フェーズ1）

- 準拠: docs/requirements.md §1（判定基準・ウェイトは本書に転記。数値の正は requirements.md）
- 採点者: 秘書（claude-in-chrome で netkeiba を閲覧・転記）
- 原則: **判定に必要な情報が読めない項目は 0 点（中立）**。独自解釈で埋めない。離散値のみ・中間値は使わない。

## 1. 閲覧手順（1レースあたり）

1. **出馬表**: `https://race.netkeiba.com/race/shutuba.html?race_id={id}`
   - 読む: 枠番・馬番・斤量・騎手・頭数・クラス・コース（A/B/C項の材料）
2. **馬柱（出馬表の各馬の近走欄）**: 同ページの近走5走
   - 読む: 前走の着順・着差・位置取り・コメント（E項）、馬場状態別の着順（F項）、同コース実績（G項）
3. **予想オッズ**: `https://race.netkeiba.com/odds/index.html?race_id={id}&type=b1`
   - 読む: 単勝人気順位（H項の予想人気）。※auto.js の odds ブロックに取得済みならそちらを使ってよい
4. 騎手リーディング順位: netkeiba「リーディングジョッキー」ページ（当年）。週初に1回確認してキャッシュでよい（D項）

所要目安: 1レース 3〜5分（12R×3場 ≒ 2〜3時間。参戦見込みレースを優先し、ルールフィルタ落ち（新馬/未勝利/7頭以下）は A〜G を簡略化してよいが S は必ず付ける）。

## 2. 判定基準（requirements §1-2 の転記）

`S = 50 + A+B+C+D+E+F+G+H` → 0〜100 にクリップ。各項目は上から最初に該当した行を採用。

| 項目 | 判定 | 加点 |
|---|---|---|
| A. 枠適性 | 有利枠 / やや有利 / 中立 / やや不利 / 不利枠 | +8 / +4 / 0 / -4 / -8 |
| B. 脚質適性 | 合致 / やや合致 / 中立・不明 / やや不利 / 不利 | +10 / +5 / 0 / -5 / -10 |
| C. 斤量(前走比) | -2.0kg以上減 / -1.0〜-1.5 / ±0.5以内 / +1.0〜+1.5 / +2.0以上増 | +5 / +3 / 0 / -3 / -5 |
| D. 騎手 | 当年1〜10位 / 11〜30位 / 31位以下 / 上位→31位以下へ乗替 | +6 / +3 / 0 / -3 |
| E. 前走内容 | 勝ちor0.2秒内 / 0.3〜0.5秒 / 不利明記の負け / 掲示板(0.6秒以上) / 6着以下(1.0秒未満) / 1.0秒以上大敗 | +10 / +6 / +5 / +3 / -5 / -10 |
| F. 馬場適性 | 想定馬場で3着内あり / 実績なし・不明 / 想定馬場6着以下のみ(2走以上) | +6 / 0 / -6 |
| G. 同コース実績 | 勝ちあり / 3着内あり / 初・実績なし / 6着以下のみ(2走以上) | +5 / +3 / 0 / -5 |
| H. 人気乖離 | 下記の2パス方式 | -6〜+6（暫定。§H参照） |

### E採点時の必須確認事項【2026-07-05追加・calibration-review §2】
E=-5または-10（着差ベースの負け判定）を付ける前に、**必ず該当レースのレース脚質コメント・上がり3F順位を確認し、不利（出遅れ・詰まり・エンジン不利等）の有無を判定してから加点行を選ぶこと**。着順・着差だけを見て機械的に-5/-10を付けない。不利が明記されている大敗は「不利明記の負け→+5」の行を優先適用する。
（背景: 直近1走の着順のみで判定すると、不利で着を落とした馬を実力以下に評価してしまいノイズに弱い。函館10R⑦が前走16着大敗の一発評価のみでE=-10としたが実際は3着好走した事例あり）

### H の2パス方式（§1-3）【2026-07-05・レンジ暫定圧縮】
1. H抜きの素点 S0 = 50+A+…+G を全馬計算し、S0 降順で構造順位を付ける。
2. 乖離 = 予想人気順位 − 構造順位。
3. **暫定**: 乖離 ≥+4→+6 ／ +2〜+3→+3 ／ -1〜+1→0 ／ -2〜-3→-3 ／ ≤-4→-6。
   （原案は±10。A(枠適性)/C(斤量)/D(騎手)が未実装で0点固定の間、S0のレンジが狭く縮んでいるのにHだけフルレンジで効くと過剰減点になるための暫定圧縮。A/C/D実装後に±10へ復帰。詳細: docs/calibration-review-2026-07-05.md §1）
4. 予想オッズ未取得なら H=0 とし、レースに `"hPending": true`。

## 3. 出力形式（ev.mjs の入力JSON）

1レース1オブジェクト。配列で複数レースまとめてよい。

```json
{
  "raceId": "202602010511",
  "venue": "函館", "no": "11R", "name": "青函S (OP)",
  "cls": "OP/重賞", "field": 14,
  "hPending": false,
  "scores": { "1": 66, "2": 48, "12": 78 },
  "odds": {
    "win":   { "1": 21.0, "12": 4.6 },
    "place": { "1": [3.9, 5.6], "12": [1.6, 2.1] }
  },
  "boxNums": [12, 3, 6, 5, 1]
}
```

- `field` は**必須**（出走頭数）。欠落すると複勝の正規化が k=2（7頭以下扱い）に落ち、複勝EVが過小になる。
- `scores`: 全出走馬の S（馬番キー）。odds は auto.js の `odds.races[raceId]` からコピー。
- `boxNums`: 5頭BOXの馬番（省略時はスコア上位5頭が自動選定される）。
- 実行: `node scripts/ev.mjs scores.json` → data.js 用の races/scan 断片JSONが出力される。
- 出力の `horses` から box 5頭分の score/pWin/pPlace/evWin/evPlace を data.js の `box[]` へ、`sanrenpuku`/`evLabel`/`call` をレースへ、`scan` 断片を scan 行へ転記する。

## 4. 注意

- ウェイト・閾値は仮決め（K1〜K3）。**勝手に変えない**。違和感はメモに残してフェーズ2の較正で扱う。
- フェーズ1は全レース `provisional: true`（UIに「暫定」印が出る）。
- SLA: 前日 21:00 までに当該日の全レース分を data.js に反映（requirements §8 S2）。
